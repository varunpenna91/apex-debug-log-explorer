# Apex Debug Log Explorer v0.1.0

![Apex Debug Log Explorer execution graph](../media/hero-execution-graph-dark.png)

## What It Does

Apex Debug Log Explorer turns Salesforce Apex debug logs into an interactive execution graph. Instead of scrolling through thousands of raw log lines, you can follow the transaction from Apex entry point to DML, triggers, Flow interviews, SOQL, Async Apex requests, email sends, callouts, and exceptions.

## Who It Helps

- Salesforce developers troubleshooting complex logs.
- Production support engineers triaging failed transactions.
- Salesforce architects reviewing DML/SOQL paths and automation impact.
- New team members trying to understand how a transaction traveled through Apex, triggers, and Flow.

## Core Capabilities

- Graph-based transaction view with expandable downstream execution.
- DML, SOQL, Errors, Email, and Callouts indexes that jump to exact graph nodes.
- Grouped SOQL/DML rows that can highlight every matching occurrence.
- Flow interview and Flow element distinction.
- Trigger, Flow, Error, Apex Action, Async Apex, and Callout graph filters.
- Raw evidence drawer showing the original log lines behind the selected node.
- Light/dark mode and first-run guided product tour.
- Local parsing with no Salesforce login and no server upload.

## How It Differs From Existing Tools

Salesforce recommends Apex Log Analyzer as a useful extension for understanding debug logs, and the Marketplace listing describes Certinia Apex Log Analyzer as strong for flame charts, call trees, SOQL/DML summaries, execution timelines, raw-log navigation, and event filtering. Apex Debug Log Explorer is intentionally different: it is built around graph-first cause-and-effect navigation through a Salesforce transaction.

Salesforce Apex Replay Debugger is a debugging workflow for stepping through Apex with debugger-oriented setup. Apex Debug Log Explorer is an after-the-fact visual triage tool for logs you already have.

The main product question this release answers is:

> Where did this DML, SOQL, email, callout, or exception happen, what invoked it, and what did it trigger downstream?

## Technical Notes

- React + TypeScript + Vite frontend.
- Electron desktop shell.
- VS Code webview extension package.
- Web Worker parser with main-thread fallback for restricted webview environments.
- Directed graph layout with incremental expansion.
- No Salesforce auth, no server backend, no log upload.

## Install: Desktop App

### macOS

1. Download the right `.dmg` for your Mac:
   - Apple Silicon: `apex-debug-log-explorer-0.1.0-mac-arm64.dmg`
   - Intel Mac: `apex-debug-log-explorer-0.1.0-mac-x64.dmg`
2. Open the `.dmg`.
3. Drag **Apex Debug Log Explorer** to Applications.
4. Launch the app and open a Salesforce debug `.log` or `.txt` file.

### Windows

1. Download `apex-debug-log-explorer-0.1.0-win-x64.exe`.
2. Run the installer.
3. Launch **Apex Debug Log Explorer** from the Start menu or desktop shortcut.
4. Open a Salesforce debug `.log` or `.txt` file.

Initial desktop builds are unsigned. macOS may show a Gatekeeper warning, and Windows may show Microsoft Defender SmartScreen until signed builds have reputation. If your organization requires signed and notarized apps, build from source or sign the artifacts internally.

## Install: VS Code Extension

### Option 1: VS Code Marketplace

1. Open Extensions in VS Code.
2. Search for `Apex Debug Log Explorer`.
3. Click Install.

Command-line install:

```bash
code --install-extension apex-debug-log-explorer.apex-debug-log-explorer
```

Marketplace page:

https://marketplace.visualstudio.com/items?itemName=apex-debug-log-explorer.apex-debug-log-explorer

### Option 2: Install From VSIX

1. Download `apex-debug-log-explorer-0.1.0.vsix` from this release.
2. Open VS Code.
3. Open Command Palette.
4. Run `Extensions: Install from VSIX...`.
5. Select the downloaded `.vsix`.
6. Open a Salesforce debug `.log` file.
7. Run `Apex Debug Log Explorer: Open Current File`.

VSIX command-line install:

```bash
code --install-extension apex-debug-log-explorer-0.1.0.vsix
```

After installation:

- Right-click a `.log` or `.txt` file and choose `Open with Apex Debug Log Explorer`.
- Or run `Apex Debug Log Explorer: Open Log` from Command Palette.

## Run From Source

```bash
npm install
npm run desktop
```

For browser/dev mode:

```bash
npm run dev -- --port 5173
```

For VS Code extension packaging:

```bash
npm run vscode:package
```

## Known Limitations

- The visualization depends on what Salesforce emitted in the debug log.
- Truncated logs can miss downstream details.
- This release does not include AI suggestions.
- This release does not connect directly to Salesforce orgs.
- Initial desktop builds are unsigned.
